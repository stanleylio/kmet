"Stale"?

at least two types of "stale":

  • logger not getting new reading (e.g. sensor down, logger process crashed)
  • connection to web server lost

real-time plots check the first kind; windpolar checks the second kind

rationale: report mV, or report W/m^2? the former is useless for visualization and debugging, but the latter involves post-processing which involves calibration factor and conversion formula (approximation). these are assumptions and factors that change all the time. there is no way to enforce that "marine tech shall update the calibration factor every time a sensor is swapped"
